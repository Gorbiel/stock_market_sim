# Contributing

## Scope

This repository was created as part of a 2026 backend engineering recruitment task.  
Contributions are not expected, but the guidelines below reflect the intended workflow.

## Branching

- `master` → protected branch
- `feature/*` → new features or tasks
- `fix/*` → bug fixes
- `refactor/*` → code improvements

## Commits

Use clear and descriptive commit messages:

- `feat: add order endpoint`
- `fix: handle insufficient stock in wallet`
- `refactor: simplify service logic`

## Pull Requests

- Keep PRs small and focused
- Ensure the project builds successfully
- Ensure tests pass
- Follow existing code structure and conventions

## General Guidelines

- Prefer readability over cleverness
- Keep business logic in the service layer
- Avoid unnecessary complexity
